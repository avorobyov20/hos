О сайте

Spacemusic.space является приложением к моему резюме на hh.ru


Цель создания сайта

Подтвердить свой опыт в качестве web-разработчика на Django Framework.


Что именно реализовано

Создан клон сайта еженедельной американской радиопередачи Hearts of Space, содержащий первые 1090 эфиров, каждый из которых представлен отдельным плейлистом.


Возможности гостевого режима

•  Навигация по категориям

•  Поиск по названию эфира

•  Прослушивание любого эфира целиком или отдельными треками


Что дает регистрация

Каждый зарегистрированный пользователь получает:

•  возможность сохранять плейлисты эфиров себе в "Избранное"

•  четыре пустых собственных плейлиста, которые можно наполнять понравившимися треками из разных эфиров


История создания

Это вторая версия сайта. Первая была реализована средствами стандартного приложения Django Admin. Переключиться в "Django Admin-версию" можно и сейчас через меню Профиль -> Мои плейлисты.

Основным недостатком первой версии была нагрузка на сервер т.к. страницы полностью перегружались почти при каждом клике.

Сейчас страница перегружается только если пользователь решит "запостить" изменения в свой профиль (ajax радикально снизил нагрузку и время отклика).

"Django Admin-версия" пока сохранена, т.к. она имеет гараздо более мощные средства поиска. Например, позволяет найти все незащищенные системой Content ID треки, относящиеся к определенному музыкальному стилю.


Что можно было бы улучшить

В порядке значимости

•  Вместо мгновенного, но посимвольного поиска по названию применить релевантный Elasticsearch по описанию эфира

•  Добавить возможность превращать собственные плейлисты в подкасты для прослушивания в подкаст-плеерах

•  Добавить rest api (если кто-то захочет перенести приложение на другие платформы, т.к. сейчас весь интерфейс заточен на десктопные браузеры и управление мышью)

•  Использовать Memcached для хранения содержимого плейлистов (возможно, это сократит на сколько-то миллисекунд отклик при переключении между листами)

•  Залить недостающие эфиры 1091-1327

•  Перенести из "админки" функционал, связанный с переименованием пользовательских плейлистов и переупорядочиванием треков внутри них
